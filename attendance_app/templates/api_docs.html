🚀 Getting Started

Welcome to the JD TECH Attendance System Developer API. This portal provides comprehensive JSON endpoints for integrating third-party systems, mobile client applications, SCADA platforms, and automation tools with the corporate attendance ledger.

All API requests are expected to send and receive standard application/json payloads unless specified otherwise. Requests must be authenticated either through active web session cookie context (for front-end integrations) or via an authorization Bearer token.

🔐 Token Authentication

The API employs secure, signed cryptographic tokens using the URLSafeSerializer schema. Standard integration tokens can be generated below by authenticating with your system credentials, or programmatically via the POST /api/auth/login endpoint.

🔑 Sandbox Credential Manager
STATELESS JWT

Log in with your existing account credentials to obtain an API key/token. The token will be securely saved in your local browser session and automatically injected into the headers of all interactive API test console queries below.

Active Bearer Token No active token
POST /api/auth/login Request signed API token with user credentials
▼
📋 Payload Specification
username* string System username
password* string User account password
💻 Code Snippet
curl -X POST {{ network_url }}/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "yourpassword"}'fetch('{{ network_url }}/api/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ username: 'admin', password: 'yourpassword' })
})
.then(res => res.json())
.then(data => console.log(data));import requests

url = "{{ network_url }}/api/auth/login"
payload = {"username": "admin", "password": "yourpassword"}

response = requests.post(url, json=payload)
print(response.json())

👥 Employee Directory

These endpoints manage the company's active employees roster. Administrative roles can create or remove employee records directly. Regular employee accounts are allowed read-only queries of active directories.

GET /api/employees List active or inactive employees
▼
📋 Query Parameters
active string Optional. Filter by status: true or false.
💻 Code Snippet
curl {{ network_url }}/api/employees?active=true \
  -H "Authorization: Bearer YOUR_TOKEN"fetch('{{ network_url }}/api/employees?active=true', {
  method: 'GET',
  headers: { 'Authorization': 'Bearer YOUR_TOKEN' }
})
.then(res => res.json())
.then(data => console.log(data));import requests

url = "{{ network_url }}/api/employees"
headers = {"Authorization": "Bearer YOUR_TOKEN"}
params = {"active": "true"}

response = requests.get(url, headers=headers, params=params)
print(response.json())
⚡ Interactive Sandbox Client
Filter (active)
Response Output HTTP 200
POST /api/employees Add a new employee to the database
▼
📋 Payload Specification
employee_id* string Unique identifier (e.g. EMP027)
full_name* string Employee full name
department* string Department (e.g. SCADA, PLC, IT)
position string Job role/title
email string Email address
phone string Phone contact
join_date string ISO format YYYY-MM-DD. Defaults to today.

curl -X POST {{ network_url }}/api/employees \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \

  method: 'POST',
  headers: { 
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_TOKEN'
  },
  body: JSON.stringify({
    employee_id: 'EMP100',
    full_name: 'Ramesh Kumar',
    department: 'SCADA',
    position: 'Engineer'
  })
})
.then(res => res.json())
.then(data => console.log(data));import requests

url = "{{ network_url }}/api/employees"
headers = {
    "Authorization": "Bearer YOUR_TOKEN",
    "Content-Type": "application/json"
}
payload = {
    "employee_id": "EMP100",
    "full_name": "Ramesh Kumar",
    "department": "SCADA",
    "position": "Engineer"
}

response = requests.post(url, headers=headers, json=payload)

📅 Attendance Ledger

These endpoints manage real-time employee attendance marking, check-in registration, and custom remarks. Access Control Notice: Regular employee tokens can query or mark only their own personal attendance. Administrative tokens can query, add, or overwrite any employee's ledger.

GET /api/attendance Fetch filtered list of attendance records
▼
📋 Query Parameters
employee_id string Filter by employee. (Ignored for non-admins).
date string ISO format YYYY-MM-DD. Filter by single day.
start_date string ISO format YYYY-MM-DD. Filter range start.
end_date string ISO format YYYY-MM-DD. Filter range end.
department string Filter by company department.
💻 Code Snippet
curl "{{ network_url }}/api/attendance?date=2026-05-26" \
  -H "Authorization: Bearer YOUR_TOKEN"fetch('{{ network_url }}/api/attendance?date=2026-05-26', {
  method: 'GET',
  headers: { 'Authorization': 'Bearer YOUR_TOKEN' }
})
.then(res => res.json())
.then(data => console.log(data));import requests

url = "{{ network_url }}/api/attendance"
headers = {"Authorization": "Bearer YOUR_TOKEN"}
params = {"date": "2026-05-26"}

response = requests.get(url, headers=headers, params=params)
print(response.json())

POST /api/attendance/mark Register or update check-in ledger details
▼
📋 Payload Specification
employee_id string Target employee code. Optional for self-marking. Mandatory for admins.
date string ISO format YYYY-MM-DD. Defaults to today's date.
status* string Must be one of: Present, Absent, or Half Day.
check_in string HH:MM format. Defaults to local server current time.
check_out string HH:MM format. Register checkout details.
location string Active location site name (e.g. MRF TC1). Set to empty if Absent.
notes string Custom remarks or logs.
💻 Code Snippet
curl -X POST {{ network_url }}/api/attendance/mark \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"status": "Present", "location": "MRF TC1", "notes": "On-site"}'fetch('{{ network_url }}/api/attendance/mark', {
  method: 'POST',
  headers: { 
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_TOKEN'
  },
  body: JSON.stringify({
    status: 'Present',
    location: 'MRF TC1',
    notes: 'On-site installation completed'
  })
})
.then(res => res.json())
.then(data => console.log(data));import requests

headers = {
    "Authorization": "Bearer YOUR_TOKEN",
    "Content-Type": "application/json"
}

    "status": "Present",
    "location": "MRF TC1",
    "notes": "On-site installation completed"

response = requests.post(url, headers=headers, json=payload)
